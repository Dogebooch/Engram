# SAM-assisted outlines (sam_segment.py) — install INTO the same .venv-video-ingest
# AFTER tools/video-ingest/requirements.txt.
#
# torch + torchvision must come from the AMD ROCm index FIRST (they are not on
# PyPI for gfx1100). Per the project's hardware notes:
#
#   .\.venv-video-ingest\Scripts\python.exe -m pip install --pre torch torchvision \
#     --index-url https://rocm.nightlies.amd.com/v2/gfx110X-dgpu/
#
# Then:  .\.venv-video-ingest\Scripts\python.exe -m pip install -r tools\video-ingest\requirements-sam.txt
#
# NOTE (verified 2026-05-28, ROCm 7.10 nightly / RX 7900 XTX): SAM-family
# attention is numerically broken / crashes on the GPU here (garbage masks, IoU
# scores like 13073) — confirmed for BOTH sam2 and mobilesam. So segmentation
# runs on CPU (--device cpu). The DEFAULT backend is MobileSAM: ~2s encode on
# CPU, instant predicts, tight masks — fast enough for bulk. SAM2 (--backend
# sam2 --model large) is a slower quality-escalation option. Re-test --device
# cuda when ROCm flash attention stabilises for gfx1100.
